ShopBot escalates to a human agent via 4 trigger types. Each trigger auto-packages session context so agents receive a full handoff summary — no customer has to repeat themselves.

TRIGGER_01 · Sentiment
Negative sentiment detected
If the engine detects anger or distress across ≥2 consecutive messages, or sentiment score drops below -0.60, the bot acknowledges empathetically and offers human escalation proactively.
sentiment_score < -0.60
TRIGGER_02 · Retry Limit
3 failed clarification attempts
If a user fails to provide a required slot (e.g. order ID) after 3 prompts, the bot stops asking and routes to a human who can look up the account manually via CRM.
slot_retry_count ≥ 3
TRIGGER_03 · High Stakes
High-value or legal issue
Orders above $500, refund disputes, legal threats, or fraud mentions always escalate immediately — regardless of sentiment score or confidence level.
order_value > $500 OR legal_flag
TRIGGER_04 · Manual
User requests human directly
Any message matching the human_handoff intent (confidence ≥ 0.65) immediately routes to an agent with the full conversation transcript attached to the ticket.
intent == human_handoff
